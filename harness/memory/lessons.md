# Harness Lessons & Memory

这里记录了长期开发、验证过程中获得的经验教训，作为智能体环境系统的“记忆”。

## Examples

### [1. 依赖冲突与打包修复]
当且仅当发生 Maven Checkstyle 或者 ArchUnit 插件冲突时：
1. 请勿直接删除整个 maven-plugins 或注释掉它们。
2. 转而在 pom.xml 的配置段排查并添加 exclusions。

### [2. 针对复杂多字节文件 (README.md) 的修改失败及其解决方案]
在对包含大量中文字符、表格、Mermaid 图表的文件进行 `replace_file_content` 时，可能会因编码或格式匹配问题导致失败。此时不应在对话中盲目尝试多次修改，以免浪费上下文缓存，而应：
1. **优先使用脚本修复**：通过 Python 或 Shell 编写一个临时的字符串精确替换脚本来完成该任务。
2. **确认文件索引**：确认目标内容在文件中的准确位置，防止跨行匹配导致的错误。
3. **经验沉淀**：将失败的具体路径和特征记录在 `harness/trace/` 中，防止下一个 Agent 重蹈覆职。
